# Data

## Committed sample

`data/sample/scan_32.npz` is a fully synthetic 32x32-position 4D-STEM scan
(64x64-pixel patterns, 6 grains, dose 300 electrons per pattern) generated by
this repository's own simulator with a fixed seed:

    orient4d simulate --scan 32 --grains 6 --dose 300 --seed 42 --out data/sample/scan_32.npz

It contains exact per-position ground truth (grain id, phase, local
orientation, purity) alongside the Poisson count data, so the demo and the
tutorial notebook run and score with no download. No measured data of any
kind is included; everything is simulated.

## Bring your own 4D-STEM data

No experimental dataset is committed, because openly licensed raw 4D-STEM
scans are rare and licensing is often unclear. The loader accepts any 4D
array of shape `(scan_rows, scan_cols, det_rows, det_cols)` saved as `.npy`,
or as `.npz` under the key `data`:

    from orient4d import load_external
    data, det = load_external("my_scan.npy", k_max=1.4)

`k_max` is the detector half-width in reciprocal angstrom for your camera
length; it sets the virtual-image radii and the scale of any template
library you build. Convert vendor formats with the usual tools, for example:

    # HyperSpy (.dm4, .hspy, ...)
    import hyperspy.api as hs
    s = hs.load("scan.dm4")
    import numpy as np
    np.save("my_scan.npy", s.data.astype("float32"))

    # py4DSTEM (.h5 from many acquisition systems)
    import py4dstem
    datacube = py4dstem.import_file("scan.h5")
    np.save("my_scan.npy", datacube.data.astype("float32"))

For real data you will want to build the template library from your own
crystal structures (see `orient4d.sim.PhaseSpec`) rather than the two
synthetic phases shipped here; the committed CNN was trained purely on this
repository's forward model and is not expected to transfer to experimental
patterns (see `models/MODEL_CARD.md`).

Keep full datasets out of the repository: put them under `data/full/`, which
is gitignored.
